# 💰 Дорожная карта: Heist & Wanted (расширение `/rob`)

> Цель: сделать ограбления коллективным мета-геймом, открыть доступ к банку, добавить контент-цикл «преступник ↔ жертва ↔ охотник».

---

## 📋 Этапы

### Этап 1 — Фундамент БД и магазин защиты ✅
- [x] Prisma: `EconomyHeist`, `EconomyHeistMember`, поля в `EconomyProfile` (`wantedStars`, `wantedExpiresAt`, `safeUntil`, `dirtyAmount`, `dirtyClearAt`, `lockpickReady`)
- [x] Конфиг: `EconomyConfig` — блоки `heist*`, `wanted*`, `dirty*`, `safe*` с дефолтами
- [x] Миграция через `prisma db push` (в `entrypoint.sh` при старте контейнера)
- [x] Магазин: предметы `safe` (иммунитет/-50% добычи), `lockpick` (+бонус к /rob, расходник)

### Этап 2 — `/heist` (налёт на банк) ✅
- [x] Команда `/heist target:@user` — инициирует налёт
- [x] Публичная панель «Соберите команду» с кнопками `🤝 Присоединиться` / `🚨 Начать!` / `❌ Отменить`
- [x] Минимум 2 соучастника (всего 2–4 человек, конфигурируется)
- [x] Кулдаун **24ч** на инициатора + **12ч** на участника
- [x] Шансы: base 60% + 15% за каждого сверх минимума (-20% если у жертвы сейф)
- [x] Успех: тащат **8–15% из банка**, делится поровну, остаток инициатору
- [x] Провал: штраф каждому (1000 ₪ по умолчанию) + 1 звезда розыска каждому
- [x] Атомарная транзакция, лок инициатора через `withFinancialLock`
- [x] Окно сбора команды: **5 минут**, фоновый `economy:heistResolve` шедулер раз в 30с

### Этап 3 — Wanted-система ✅
- [x] При успешном `/rob` или `/heist`: грабитель получает +1 звезду розыска (`wantedStars`)
- [x] Звёзды распадаются по 1 за **48ч** (фоновый `economy:wantedDecay` раз в час)
- [x] Команда `/capture user:@robber` — доступна только если у цели ≥3 звёзд
- [x] Capture: шанс 70%, при успехе **снимает 1 звезду** + забирает **20% кошелька** в свой
- [x] При провале capture: штраф охотнику (750 ₪) + кулдаун 6ч
- [x] Жертва изначального ограбления получает **+50%** к шансу capture (по транзакциям ROB_VICTIM/HEIST_VICTIM за 7 дней)
- [x] `/wanted top` — публичная доска розыска (топ-10 со звёздами)
- [x] `/wanted me` — посмотреть свой статус + время до следующего decay

### Этап 4 — «Сейф» и «Отмычка» (магазин) ✅
- [x] `safe`: конфигируемые цена/длительность, `safeUntil` в профиле, режимы `immune`/`partial` (пер гильдии)
- [x] `lockpick`: расходник, +bonus% к successRate следующего `/rob`, `lockpickReady` в профиле, атомарный `consumeLockpick`
- [x] Подкоманды `/shop safe` и `/shop lockpick`
- [x] Логика расхода в `/rob` и проверка safe в `/rob` + `/heist`

### Этап 5 — «Грязные деньги» ✅
- [x] Поле `dirtyAmount` + `dirtyClearAt` в профиле
- [x] При успешном грабеже/heist украденное помечается как dirty на 24ч
- [x] При попытке `/deposit`: блок, если сумма > (wallet − dirtyAmount)
- [x] Команда `/launder amount:N` — отмыть с комиссией 30%
- [x] Шедулер `economy:dirtyExpire` раз в 10 мин — авто-чистка просроченных

### Этап 6 — Мемная лента + локали ✅
- [x] Канал-лог через news-систему: random-фраза по исходу (`pickPhrase`)
- [x] **30+ ключей в локалях** `economy.cmd.dirty.*` / `economy.cmd.launder.*` / `economy.news.*`
- [x] Отдельный флавор `news_big_steal` при куше ≥ 10 000

### Этап 7 — Сборка и деплой ✅
- [x] `tsc` чисто
- [x] `npm run build:protected`
- [x] commit + push
- [x] SSH deploy + `prisma db push` (автоматически в `entrypoint.sh`)
- [x] Проверка в живой гильдии

## 🛠 Технические заметки

- Все балансы/шансы — **в `EconomyConfig`** (per-guild), значения по умолчанию в `constants.ts`
- Все денежные операции — через `withFinancialLock` + `prisma.$transaction` (как в текущем `/rob`)
- Кулдауны — поля `lastHeistInit`, `lastHeistMember`, `lastCapture` в `EconomyProfile`
- Wanted-decay — отдельный шедулер раз в час (использовать `SchedulerManager`)
- News-канал — переиспользовать существующий `news.ts`
