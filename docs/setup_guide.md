# Инструкция по настройке бота Bublik для нового сервера

Эта инструкция содержит пошаговое руководство для администраторов по первоначальной настройке и конфигурированию всех модулей бота Bublik.

---

## 1. Подготовка и приглашение бота

### Необходимые Intents (Намерения) в Discord Developer Portal:
Для полноценной работы бота должны быть включены следующие Privileged Gateway Intents:
- **Presence Intent** (для отслеживания активности)
- **Server Members Intent** (для приветствий и вычисления времени)
- **Message Content Intent** (для чтения сообщений команд)

### Права при приглашении:
Убедитесь, что бот имеет права Администратора (или как минимум: управление ролями, каналами, отправка сообщений, управление голосовыми каналами).

---

## 2. Первые шаги настройки

Сразу после входа бота на сервер:
1. Выберите язык интерфейса бота с помощью команды:
   `/language set locale:ru` (или `en` для английского)
2. Выполните команду `/setup` для открытия интерактивного мастера настройки, который покажет текущий статус готовности всех модулей.

---

## 3. Настройка модулей по шагам

### Шаг 3.1: Приветствия и Авто-роли (Welcome Module)
Этот модуль выдаёт начальные роли при входе участников на сервер и отправляет приветственные сообщения.

1. **Создайте каналы:** канал для приветствий (например, `#welcome`) и канал для тикетов (например, `#собеседование`).
2. **Создайте или выберите роли:** авто-роль для новоприбывших (например, `@Новобранец`), роль полноценного участника (например, `@Боец`) и роль кандидата (например, `@Кандидат`).
3. **Выполните команду настройки:**
   `/welcome setup auto_role:@Новобранец welcome_channel:#welcome`
4. **Настройте роли участника и тикеты:**
   `/welcome setup member_role:@Боец recruit_role:@Кандидат ticket_channel:#собеседование`
5. **Проверьте настройки:** `/welcome config`

---

### Шаг 3.2: Экономика и PB-тиры (Economy Module)
Модуль шекелей, ежедневных бонусов, мини-игр и выдачи ролей за нахождение в голосовых каналах (PB-тиры).

1. **Создайте каналы:** канал для новостей экономики, канал для логов (доступный только администрации) и канал для авто-обновляемого лидерборда.
2. **Выполните команду настройки каналов:**
   `/economy setup news:#новости log:#логи-экономики leaderboard:#лидерборд welcomebonus:500`
3. **Настройте PB-роли голосовой активности (до 10 уровней):**
   `/economy roles tier1:@Тир1 tier2:@Тир2 ...`
   *(Каждый тир привязан к часам в войсе: Tier 1 - 50ч, Tier 2 - 100ч, Tier 3 - 200ч и т.д. с повышающимся множителем шекелей)*.
4. **Добавьте товары в магазин ролей:**
   `/shop add role:@Роль name:Название price:1000 duration_hours:168 max_stock:10`
5. **Активируйте экономику:** `/economy toggle`

---

### Шаг 3.3: Временные Голосовые Каналы (TempVoice Module)
Создает динамические комнаты для общения при входе пользователя в канал-генератор.

1. **Создайте голосовой канал-генератор** (например, `➕ Создать канал`) и **категорию**, где будут создаваться временные комнаты.
2. **Выполните команду настройки:**
   `/voice setup channel:➕ Создать канал category:ВРЕМЕННЫЕ ВОЙСЫ`
3. **Задайте роль модератора (по желанию):**
   `/voice addrole channel:➕ Создать канал role:@Модератор`

---

### Шаг 3.4: Полковые Бои (RegBattle Module)
Управление отрядами во время проведения ПБ. Автоматически распределяет зашедших в войс игроков по отрядам по 8 человек и выдаёт роль «В отряде».

1. **Создайте:** голосовой мастер-канал (например, `⚔️ Вход в ПБ`), категорию для создания ПБ-войсов, текстовый канал для пингов ПБ (`#объявления-пб`) и войс-канал для запасных.
2. **Выполните команду настройки:**
   `/regbattle setup master:⚔️ Вход в ПБ category:ПОЛКОВЫЕ БОИ announce:#объявления-пб`
3. **Укажите дополнительные параметры:**
   `/regbattle setup reserve:ВойсЗапасных ping_role:@Бойцы insquad_role:@В отряде`
4. **Укажите роли командиров (кто имеет доступ к управлению отрядами):**
   `/regbattle addrole type:commander role:@Офицер`

---

### Шаг 3.5: Система отпусков (Vacation Module)
Позволяет игрокам уходить в отпуск (на это время с них снимаются роли прайм-тайма и ПБ, чтобы не портить статистику онлайна).

1. **Создайте каналы:** канал для отправки заявок модераторам (`#заявки-на-отпуск`) и канал логов ухода/возврата (`#логи-отпусков`).
2. **Создайте роль отпуска** (например, `@В отпуске`).
3. **Выполните команду настройки:**
   `/vacation setup review:#заявки-на-отпуск log:#логи-отпусков role:@В отпуске`
4. **Укажите роли, которые должны автоматически сниматься при уходе:**
   `/vacation addrole type:remove role:@РольПраймТайма`
5. **Укажите кураторов отпусков (кто одобряет кнопки):**
   `/vacation addrole type:reviewer role:@Офицер`
6. **Отправьте интерактивную панель отпусков в канал:**
   `/vacation panel channel:#канал-для-отпусков`

---

### Шаг 3.6: Система Команд (Teams Module)
Позволяет участникам создавать свои внутренние полковые мини-команды, подавать отчеты после тренировок и соревноваться в лидерборде.

1. **Создайте каналы:** `#заявки-на-команду`, `#отчеты-команд`, `#сборы`, `#рейтинг-команд`.
2. **Выполните команду настройки:**
   `/team setup application_channel:#заявки-на-команду report_channel:#отчеты-команд poll_channel:#сборы`
3. **Привяжите лидерборд и базовую роль доступа:**
   `/team setup leaderboard_channel:#рейтинг-команд base_role:@Боец approver_role:@Офицер`

---

### Шаг 3.7: Справочник БР (BR Module)
Справочник техники и расписание ротаций БР для War Thunder.

1. **Создайте канал** для справочника БР.
2. **Отправьте интерактивную панель:**
   `/br panel channel:#справочник-бр`
3. **Обновите расписание ротаций** (прикрепите JSON-файл):
   `/br rotation-update`
4. **Импорт техники:**
   Поскольку база данных техники содержит сотни строк, она импортируется через терминал SSH с помощью скрипта. Передайте TSV-файл в стандартный поток ввода:
   ```bash
   ts-node scripts/migrate-br.ts <guildId> < br_entries.tsv
   ```
